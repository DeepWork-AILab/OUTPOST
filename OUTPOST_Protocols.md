# Project OUTPOST: Технические Протоколы (DeepWork AILab)
**Дата последнего обновления:** 2026-04-23
**Статус:** Репозиторий инициализирован. Ядро v2.0 активно.

## 🛡️ Слой Security (Shield-24)
*Перед деплоем любого протокола сервер должен соответствовать стандарту безопасности:*
1. **ZTNA Identity:** Каждый узел генерирует `Device Trust ID` через HMAC-SHA256(machine-id). 
   - Ключ хранится в `/etc/outpost/trust.key` (права 0400).
2. **SSH Hardening:** Отключение аутентификации по паролю. Только ключи Ed25519.
3. **Firewall (UFW/Iptables):** Закрыты все порты, кроме 443 (VLESS), 4500 (AWG) и 4433 (Hysteria2).
4. **Zero-Log Policy:** Отключение системных логов, фиксирующих IP пользователей.

## ⚙️ Стек Протоколов (Hierarchy)
1. **TIER 1 (AmneziaWG):** Приоритетный UDP-туннель. Маскировка под IPsec.
2. **TIER 2 (Hysteria2):** Агрессивный UDP/QUIC для пробития троттлинга.
3. **TIER 3 (VLESS-Reality):** TCP-резерв. Мимикрия под TLS-трафик крупных вендоров (Samsung/Amazon).

## 🐕 Failover Watchdog v2.0
*Логика автономного выживания:*
- Проверка не через ICMP, а через прикладной TCP-хендшейк.
- **UDP-Quarantine:** Если UDP-трафик блокируется ТСПУ, узел автоматически уходит на TIER 3 (TCP) на 20 минут.
- **Persistent State:** Состояние переключений сохраняется в `/var/lib/outpost/state.json`.
